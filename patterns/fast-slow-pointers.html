Cycle detection and middle finding

üéØ Key Skills to Master:
  • Cycle detection: fast pointer moves 2 steps, slow moves 1 step
  • Find middle element: when fast reaches end, slow is at middle

  • Apply to arrays: fast/slow works on any sequence with sequential access

üß† 2. Core Concept Theory
Tortoise and Hare

The Fast & Slow Pointers pattern uses two pointers moving at different speeds (usually 1x and 2x). When there is a cycle in the structure, the fast pointer will eventually meet the slow pointer.

Use when:

  • Detecting cycles in linked lists or arrays.
  • Finding the middle of a linked list.
  • Finding the start of a cycle.
üéØ 3. Real-World Amazon Scenario
Circular Dependency Detection

Scenario: In a distributed system or build system, tasks may depend on each other. If Task A depends on B, and B depends on A, we have a cycle (deadlock). Fast & Slow pointers can detect these cycles in dependency graphs (modeled as linked lists for simple chains).

üß© 5. Canonical Amazon Problem
Linked List Cycle (LeetCode 141)

Problem: Given head, the head of a linked list, determine if the linked list has a cycle in it.

Input: head = [3,2,0,-4], pos = 1
Output: true

üî¥ 6. Brute Force Analysis
Hash Set for Visited Nodes

Store every node in a Hash Set. If we see a node again, it's a cycle.

Why Amazon Rejects: It uses O(N) space. The Fast & Slow pointer approach uses O(1) space.

üü¢ 7. Optimized Solution
func hasCycle(head *ListNode) bool {
    slow, fast := head, head
    
    // Fast moves 2 steps, Slow moves 1 step
    for fast != nil && fast.Next != nil {
        slow = slow.Next
        fast = fast.Next.Next
        
        // If they meet, there is a cycle
        if slow == fast {
            return true
        }
    }
    
    return false
}

public class Solution {
    public boolean hasCycle(ListNode head) {
        ListNode slow = head;
        ListNode fast = head;
        
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
            
            if (slow == fast) {
                return true;
            }
        }
        
        return false;
    }
}

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        slow = fast = head
        
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            
            if slow == fast:
                return True
                
        return False

Trace for 3 -> 2 -> 0 -> -4 -> (back to 2)

Step 1: Init Slow = 3, Fast = 3 Step 2: Move Slow -> 2 Fast -> 0 (skipped 2) Step 3: Move Slow -> 0 Fast -> 2 (skipped -4, wrapped around) Step 4: Move Slow -> -4 Fast -> -4 (skipped 0) Result: Slow == Fast (-4). Cycle Detected!
‚öôÔ∏è 9. Complexity Analysis
Time & Space

Time Complexity: O(N)

If there is no cycle, Fast reaches the end in N/2 steps. If there is a cycle, they meet within the cycle length.

Space Complexity: O(1)

We only use two pointers.

üü£ 10. Amazon Follow-up Questions
Variations

Q: How to find the start of the cycle?
A: After they meet, reset Slow to head. Move both one step at a time. They will meet at the cycle start.

Q: How to find the length of the cycle?
A: After they meet, keep Slow stationary and move Fast until it comes back to Slow, counting steps.
