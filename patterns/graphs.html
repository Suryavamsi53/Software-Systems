Pathfinding and connectivity problems

‚úì Prerequisites You Need:
  • Understanding of graph structure: vertices (nodes), edges, adjacency lists

üéØ Key Skills to Master:
  • DFS (Depth-First Search) using recursion or stack for deep exploration
  • BFS (Breadth-First Search) using queue for shortest paths/level-order
  • Cycle detection in directed and undirected graphs
  • Topological sort for dependency resolution (tasks, courses)
  • Dijkstra's algorithm for shortest paths with positive weights
  • Union-Find (Disjoint Set Union) for connectivity and components
üí° Mental Model:

"Graphs are just nodes with connections. DFS goes deep exploring one path fully, BFS explores all neighbors level-by-level. Choose based on what you need to find."

‚úÖ 1. What Amazon EXPECTS
Interviewer's Checklist
  • BFS vs DFS: Do you know when to use which? (BFS for shortest path in unweighted graphs, DFS for exhaustive search/connectivity).
  • Visited Set: Do you always remember to track visited nodes to prevent infinite loops?
  • Input Format: Can you handle Adjacency Matrix vs Adjacency List? Can you convert between them?
  • Connected Components: Can you identify isolated sub-graphs?
üß† 2. Core Concept Theory
Nodes and Edges

A Graph is a collection of nodes (vertices) and edges connecting them. Graphs can be directed/undirected and weighted/unweighted.

Traversal:

  • DFS (Stack/Recursion): Go deep. Good for "does a path exist?"
  • BFS (Queue): Go wide level-by-level. Good for "shortest path" (unweighted).
üéØ 3. Real-World Amazon Scenario
Delivery Network Connectivity

Scenario: Amazon has fulfillment centers (nodes) connected by truck routes (edges). If a snowstorm blocks some routes, can we still deliver a package from Center A to Center B?

Solution: This is a connectivity problem. Use BFS or DFS starting from A to see if B is reachable.

üß© 5. Canonical Amazon Problem
Number of Islands (LeetCode 200)

Problem: Given an m x n 2D binary grid which represents a map of '1's (land) and '0's (water), return the number of islands (connected lands).

Input: grid = [["1","1","0"],["1","1","0"],["0","0","1"]]
Output: 2

üî¥ 6. Brute Force Analysis
No Visited Tracking

Trying to traverse without marking cells as visited (or modifying the grid) leads to infinite loops or recounting the same island parts.

üü¢ 7. Optimized Solution
func numIslands(grid [][]byte) int {
    if len(grid) == 0 {
        return 0
    }
    
    rows, cols := len(grid), len(grid[0])
    count := 0
    
    // DFS function
    var dfs func(r, c int)
    dfs = func(r, c int) {
        // Boundary checks and water check
        if r < 0 || c < 0 || r >= rows || c >= cols || grid[r][c] == '0' {
            return
        }
        
        // Mark as visited (sink the island)
        grid[r][c] = '0'
        
        // Visit neighbors
        dfs(r+1, c)
        dfs(r-1, c)
        dfs(r, c+1)
        dfs(r, c-1)
    }
    
    for r := 0; r < rows; r++ {
        for c := 0; c < cols; c++ {
            if grid[r][c] == '1' {
                count++
                dfs(r, c)
            }
        }
    }
    
    return count
}

class Solution {
    public int numIslands(char[][] grid) {
        int count = 0;
        for (int i = 0; i < grid.length; i++) {
            for (int j = 0; j < grid[0].length; j++) {
                if (grid[i][j] == '1') {
                    count++;
                    dfs(grid, i, j);
                }
            }
        }
        return count;
    }
    
    private void dfs(char[][] grid, int i, int j) {
        if (i < 0 || j < 0 || i >= grid.length || j >= grid[0].length || grid[i][j] == '0') {
            return;
        }
        
        grid[i][j] = '0'; // Mark visited
        
        dfs(grid, i + 1, j);
        dfs(grid, i - 1, j);
        dfs(grid, i, j + 1);
        dfs(grid, i, j - 1);
    }
}

class Solution:
    def numIslands(self, grid: List[List[str]]) -> int:
        if not grid: return 0
        
        rows, cols = len(grid), len(grid[0])
        count = 0
        
        def dfs(r, c):
            if r < 0 or c < 0 or r >= rows or c >= cols or grid[r][c] == '0':
                return
            
            grid[r][c] = '0' # Mark visited
            
            dfs(r+1, c)
            dfs(r-1, c)
            dfs(r, c+1)
            dfs(r, c-1)
            
        for r in range(rows):
            for c in range(cols):
                if grid[r][c] == '1':
                    count += 1
                    dfs(r, c)
                    
        return count

Trace for 2x2 grid: [['1','0'], ['0','1']]

1. Loop r=0, c=0. Found '1'. count = 1 Call DFS(0,0). Mark (0,0) as '0'. Check neighbors (0,1), (1,0)... all '0' or out of bounds. DFS returns. 2. Loop r=0, c=1. '0', skip. 3. Loop r=1, c=0. '0', skip. 4. Loop r=1, c=1. Found '1'. count = 2 Call DFS(1,1). Mark (1,1) as '0'. Check neighbors... DFS returns. Result: 2
‚öôÔ∏è 9. Complexity Analysis
Time & Space

Time Complexity: O(M * N)

We visit each cell at most once (or a constant number of times).

Space Complexity: O(M * N)

In the worst case (all land), the recursion stack can go as deep as the number of cells.

üü£ 10. Amazon Follow-up Questions
Variations

Q: What if we can't modify the grid?
A: Use a separate visited boolean matrix. Costs O(M*N) extra space.

Q: What if the grid is infinite?
A: You can't use a matrix. Use a Hash Set to store coordinates of visited '1's.

Q: How to handle diagonal connections?
A: Add 4 more recursive calls for (r+1, c+1), (r-1, c-1), etc.
