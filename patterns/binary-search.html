Binary Search Pattern

Master the O(log n) search technique for sorted datasets.

‚úÖ 1. What EXPECTS
Interviewer's Checklist
  • Correct Template: Do you use lo <= hi or lo < hi correctly?
  • Midpoint Calculation: Do you avoid integer overflow? (lo + (hi-lo)/2)
  • Termination: Does your loop terminate for 2-element arrays?
  • Search Space: Can you identify the search space even if it's not an explicit array (e.g., answer range)?
üß† 2. Core Concept Theory
Divide and Conquer

Binary Search works by repeatedly dividing the search interval in half. If the value of the search key is less than the item in the middle of the interval, narrow the interval to the lower half. Otherwise, narrow it to the upper half.

Key Recognition Signals:

  • Input is sorted (array, matrix).
  • Problem asks for O(log n) time complexity.
  • You need to find a specific value, boundary, or insertion point.
üéØ 3. Real-World Software Systems Scenario
Log Search & Version Control

Scenario: You have a build system where version #500 is good, but version #550 is broken. You need to find the first bad commit.

Why Binary Search?

  • Checking every version (Linear Scan) is too slow if running tests takes time.
  • Versions are sorted (1, 2, 3...).
  • Binary search minimizes the number of test runs to find the culprit.

üß© 5. Canonical Software Systems Problem
Binary Search (LeetCode 704)

Problem: Given an array of integers nums which is sorted in ascending order, and an integer target, write a function to search target in nums. If target exists, then return its index. Otherwise, return -1.

Input: nums = [-1,0,3,5,9,12], target = 9
Output: 4

üî¥ 6. Brute Force Analysis
Linear Scan

Iterating through the array one by one.

Time Complexity: O(n)

Why Software Systems Rejects: If the input size is 1 billion, O(n) takes seconds/minutes. O(log n) takes ~30 operations. The difference is massive at scale.

üü¢ 7. Optimized Solution
func search(nums []int, target int) int {
    // Initialize pointers
    left := 0
    right := len(nums) - 1

    // Standard template: lo <= hi
    for left <= right {
        // Prevent integer overflow
        // Equivalent to (left + right) / 2 but safer
        mid := left + (right - left) / 2

        if nums[mid] == target {
            return mid // Found
        } else if nums[mid] < target {
            // Target is in the right half
            left = mid + 1
        } else {
            // Target is in the left half
            right = mid - 1
        }
    }

    return -1 // Not found
}

class Solution {
    public int search(int[] nums, int target) {
        int left = 0;
        int right = nums.length - 1;

        while (left <= right) {
            // Prevent integer overflow
            int mid = left + (right - left) / 2;

            if (nums[mid] == target) {
                return mid;
            } else if (nums[mid] < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }

        return -1;
    }
}

class Solution:
    def search(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums) - 1
        
        while left <= right:
            # Integer division in Python
            mid = (left + right) // 2
            
            if nums[mid] == target:
                return mid
            elif nums[mid] < target:
                left = mid + 1
            else:
                right = mid - 1
                
        return -1

‚öôÔ∏è 9. Complexity Analysis
Time & Space

Time Complexity: O(log n)

In each step, we cut the search space in half. For n elements, we can divide by 2 at most log2(n) times.

Space Complexity: O(1)

We only use a few variables (left, right, mid) for pointers. No extra data structures are allocated.

üü£ 10. Software Systems Follow-up Questions
Variations

Q: What if the array has duplicates?
A: Standard binary search finds any instance. To find the first/last, you need to continue searching in the remaining half even after finding the target.

Q: What if the array is rotated?
A: You need to check which half is sorted (left or right) and adjust logic. (See LeetCode 33).

Q: Search in an infinite stream?
A: You can't set right = len-1. Instead, expand range exponentially (1, 2, 4, 8...) until you bound the target, then binary search.
