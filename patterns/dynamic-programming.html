Dynamic Programming Pattern

Optimal substructure and memoization

üîç How to Spot This Pattern: The 3-Prong Detection System
üìä INPUT SIGNAL
  • Sequence/array of choices
  • Need optimal selection
  • Subproblems exist
  • Multiple paths to solve
‚ùì QUESTION SIGNAL
  • Maximum/minimum value
  • Ways to achieve goal
  • Best sequence
  • Optimal decision
‚ö° CONSTRAINT SIGNAL
  • Overlapping subproblems
  • Optimal substructure
  • Memoization helps
  • Recursive structure
üìã Basic Requirements to Solve Problems
‚úì Prerequisites You Need:
  • Strong recursion skills: thinking top-down naturally
  • Ability to identify subproblems: what smaller instances can we solve?
  • Understanding overlapping subproblems: detecting repeated calculations
  • Basic array/matrix operations for table-building (bottom-up)
üéØ Key Skills to Master:
  • Identify optimal substructure: does optimal solution build from optimal subproblems?
  • Memoization: cache recursive results using hash map or array
  • Tabulation: build solutions bottom-up using DP table (iterative)
  • Define state: what variables uniquely identify a subproblem?
  • State transitions: how does current state relate to previous states?
  • Space optimization: sometimes only need previous row/column of DP table
üí° Mental Model:

"DP = Recursion + Memoization. Break problem into subproblems, solve each once, store results, build answer from stored results instead of recomputing."

üß† 2. Core Concept Theory
Recursion + Cache

Dynamic Programming is just recursion where you store the results of subproblems so you don't re-calculate them.

Two approaches:

  • Top-Down (Memoization): Recursive. Easy to write. "I need F(n), so I call F(n-1)."
  • Bottom-Up (Tabulation): Iterative. Efficient. "I calculate F(0), then F(1), up to F(n)."
üéØ 3. Real-World Software Systems Scenario
Supply Chain Optimization

Scenario: You have a truck with capacity W. You have a list of packages with different values and weights. Maximize the value of packages in the truck.

Solution: This is the 0/1 Knapsack problem. DP allows us to make the optimal decision for capacity w based on the optimal decision for capacity w - weight[i].

üß© 5. Canonical Software Systems Problem
Climbing Stairs (LeetCode 70)

Problem: You are climbing a staircase. It takes n steps to reach the top. Each time you can either climb 1 or 2 steps. In how many distinct ways can you climb to the top?

Input: n = 3
Output: 3 (1+1+1, 1+2, 2+1)

üî¥ 6. Brute Force Analysis
Naive Recursion

f(n) = f(n-1) + f(n-2) without caching.

Time Complexity: O(2^n). The recursion tree grows exponentially.

Why it fails: Re-calculates the same values (e.g., f(5) calls f(3) multiple times).

üü¢ 7. Optimized Solution
func climbStairs(n int) int {
    if n <= 1 {
        return 1
    }
    
    // Space Optimized DP
    // We only need the last two values
    prev2 := 1 // step 0
    prev1 := 1 // step 1
    
    for i := 2; i <= n; i++ {
        current := prev1 + prev2
        prev2 = prev1
        prev1 = current
    }
    
    return prev1
}

class Solution {
    public int climbStairs(int n) {
        if (n <= 1) return 1;
        
        int prev2 = 1;
        int prev1 = 1;
        
        for (int i = 2; i <= n; i++) {
            int current = prev1 + prev2;
            prev2 = prev1;
            prev1 = current;
        }
        
        return prev1;
    }
}

class Solution:
    def climbStairs(self, n: int) -> int:
        if n <= 1:
            return 1
            
        prev2, prev1 = 1, 1
        
        for i in range(2, n + 1):
            current = prev1 + prev2
            prev2 = prev1
            prev1 = current
            
        return prev1

Time Complexity: O(N)

We optimized space by only storing 2 variables instead of an array of size N.

üü£ 10. Software Systems Follow-up Questions
Variations

Q: What if you can climb 1, 2, or 3 steps?
A: dp[i] = dp[i-1] + dp[i-2] + dp[i-3].

Q: What if there are costs associated with steps?
A: Min Cost Climbing Stairs. dp[i] = cost[i] + min(dp[i-1], dp[i-2]).
