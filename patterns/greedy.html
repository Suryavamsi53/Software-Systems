üéØ Key Skills to Master:
  • Verify greedy choice property: is the locally best choice globally optimal?
  • Sort effectively: choose correct sort order for the greedy strategy
  • Make greedy choice at each step: select best available option
  • Interval problems: activity selection, meeting rooms, task scheduling
  • Prove correctness: why greedy works (by exchange or induction argument)
  • Recognize non-greedy problems: when greedy fails and DP is needed

‚úÖ 1. What Amazon EXPECTS
Interviewer's Checklist
  • Greedy Choice Property: Can you explain why a local optimal choice leads to a global optimum? (Intuitive proof).
  • Sorting: Do you recognize that sorting is often the first step in greedy problems?
  • Counter-examples: Can you think of cases where greedy fails? (This shows depth).
  • Efficiency: Greedy solutions are usually O(N log N) or O(N). They expect this efficiency over DP if applicable.
üß† 2. Core Concept Theory
Local Best = Global Best

Greedy algorithms make the optimal choice at each step as they go. They never look back (no backtracking) and never look ahead (no DP).

Key Insight: If choosing the "best" option right now doesn't limit your ability to reach the best solution later, use Greedy.

üéØ 3. Real-World Amazon Scenario
Meeting Room Scheduling

Scenario: You manage a conference room at an Amazon office. You have a list of meeting requests with start and end times. You want to host as many meetings as possible.

Solution: Sort meetings by end time. Always pick the meeting that ends earliest (and doesn't overlap). This leaves the most time remaining for other meetings.

Greedy Strategy (Jump Game)

We track the maxReach at every step. If we can reach index i, we update maxReach = max(maxReach, i + nums[i]). If maxReach covers the last index, we win.

üß© 5. Canonical Amazon Problem
Jump Game (LeetCode 55)

Problem: You are given an integer array nums. You are initially positioned at the array's first index, and each element in the array represents your maximum jump length at that position. Return true if you can reach the last index.

Input: nums = [2,3,1,1,4]
Output: true

üî¥ 6. Brute Force Analysis
Backtracking (Try All Jumps)

Try every possible jump length from every position recursively.

Time Complexity: O(2^n). There are exponential paths.

Why it fails: TLE for large arrays. We re-calculate reachability for the same indices many times.

üü¢ 7. Optimized Solution
func canJump(nums []int) bool {
    maxReach := 0
    target := len(nums) - 1
    
    for i, jump := range nums {
        // If we can't reach current index i, we can't go further
        if i > maxReach {
            return false
        }
        
        // Update the farthest we can reach
        currentReach := i + jump
        if currentReach > maxReach {
            maxReach = currentReach
        }
        
        // Optimization: Early exit
        if maxReach >= target {
            return true
        }
    }
    
    return true
}

class Solution {
    public boolean canJump(int[] nums) {
        int maxReach = 0;
        for (int i = 0; i < nums.length; i++) {
            if (i > maxReach) return false;
            maxReach = Math.max(maxReach, i + nums[i]);
            if (maxReach >= nums.length - 1) return true;
        }
        return true;
    }
}

class Solution:
    def canJump(self, nums: List[int]) -> bool:
        max_reach = 0
        for i, jump in enumerate(nums):
            if i > max_reach:
                return False
            max_reach = max(max_reach, i + jump)
            if max_reach >= len(nums) - 1:
                return True
        return True

‚öôÔ∏è 9. Complexity Analysis
Time & Space

Time Complexity: O(N)

We iterate through the array once.

Space Complexity: O(1)

We only store one variable maxReach.

üü£ 10. Amazon Follow-up Questions
Variations

Q: What if we want the minimum number of jumps?
A: Use BFS-like greedy approach (Jump Game II). Track currentEnd of the current jump level and farthest reachable. Increment jumps when i reaches currentEnd.

Q: What if values are negative?
A: "Jump" usually implies forward movement. If backward is allowed, it becomes a Graph BFS problem (shortest path).
