Two Pointers Pattern

Master the most fundamental array manipulation technique in Amazon interviews

üîç How to Spot This Pattern: The 3-Prong Detection System
üìä INPUT SIGNAL
  • Sorted array or string
  • Need to find pair/triplet
  • Compare two elements
  • Reverse or rearrange
‚ùì QUESTION SIGNAL
  • Find pair that sums to target
  • Remove duplicates
  • Container with most capacity
  • Valid palindrome check
‚ö° CONSTRAINT SIGNAL
  • O(n) time complexity
  • O(1) space (no extra array)
  • In-place modification needed
  • No sorting allowed
üìã Basic Requirements to Solve Problems
‚úì Prerequisites You Need:
  • Understanding of arrays and how to access elements by index
  • Basic loop control (while loops, break/continue)
  • Comparison operators and logical conditions (==, !=, <, >, &&, ||)
  • Function return values and how to return multiple results
üéØ Key Skills to Master:
  • Moving two independent pointers left and right simultaneously
  • Deciding when to move left pointer vs right pointer based on condition
  • Recognizing problems where sorted or split arrays benefit from two pointers
  • Handling edge cases: empty arrays, single elements, duplicates
  • Writing O(1) space solutions without extra data structures
üí° Mental Model:

\"Two independent pointers exploring from different directions‚Äîwhen one pointer finds what we're looking for on its side, the other follows or adjusts accordingly.\"

‚úÖ Interviewer's Checklist

üß† 2. Core Concept Theory
üß† What The Pattern Actually Solves

Two Pointers is a technique to traverse an array/string with two references simultaneously, moving them toward each other or in the same direction based on the problem logic.

Why it works: By controlling both ends or both speeds, you can find pairs, detect cycles, or solve matching problems in a single pass.

Core insight: Instead of nested loops O(n¬≤), two pointers often solve it in O(n) with one loop.

üî∑ Recognition Keywords:
  • "Find a pair that sums to target"
  • "Container with most water"
  • "Remove duplicates in-place"
  • "Reverse a string/array"
  • "Move all zeros to end"
  • "Partition array by a value"
  • "Sorted array, find matching pairs"
üî¥ When NOT to Use Two Pointers:
  • Problem requires finding all combinations (use backtracking)
  • You need to preserve original order AND modify (use hash map)
  • Problem is on 2D matrix (use different pattern)

‚≠ê Request Rate Limiting (Real Software Systems Use Case)

Input: Array of (user_id, timestamp) pairs, already sorted by user_id, then timestamp

Why two pointers works:

  • Data is sorted by user_id ‚Üí can group with one pointer
  • For each user, compare consecutive timestamps with two pointers
  • No nested loops needed
  • O(n) time, O(1) space
Other Software Systems Real-World Examples:
  • Matching Delivery Partners: Match cheapest and fastest delivery pairs from sorted list
  • Payment Processing: Find if two transactions match (expected vs actual) in O(n) time
  • Inventory Reconciliation: Find discrepancies between two sorted inventory lists
  • User Engagement: Find pairs of timestamps from different events that correlate

üß© 5. Canonical Software Systems Problem
‚úÖ Container With Most Water (LeetCode 11 - VERY Common at Software Systems)

Problem Statement:

You are given an integer array height of length n. There are n vertical lines drawn such that the two endpoints of the i-th line are (i, 0) and (i, height[i]).

Find two lines that together with the x-axis form a container, such that the container contains the most water.

Constraints:

  • 2 ‚â§ n ‚â§ 10^5
  • 0 ‚â§ height[i] ‚â§ 10^4

Example 1:

Input: height = [1,8,6,2,5,4,8,3,7]
Output: 49
Explanation: The vertical lines are at indices 1 and 8.
Water capacity = min(8, 7) √ó (8 - 1) = 7 √ó 7 = 49

Why Software Systems Likes This:

  • Tests understanding of constraint optimization
  • Two pointers is optimal; brute force is obvious but too slow
  • Requires mathematical thinking (why move certain pointer)
  • Easy to get O(n¬≤) wrong, hard to get O(n) right without guidance
  • Good follow-up questions possible (what if inverted, rotated, etc.)
üî¥ 6. Brute Force ‚Üí Why It Fails
üî¥ Naive Nested Loop Approach
// Brute Force - WRONG for Software Systems
// Check every pair (i, j)
func maxArea(height []int) int {
    maxWater := 0
    for i := 0; i < len(height); i++ {
        for j := i + 1; j < len(height); j++ {
            // Calculate area for this pair
            width := j - i
            h := min(height[i], height[j])
            area := width * h
            if area > maxWater {
                maxWater = area
            }
        }
    }
    return maxWater
}

// Time: O(n¬≤) - For n=10^5, this is 10^10 operations!
// Space: O(1)

üî¥ Why Software Systems Rejects This:

  • Timeout: With n up to 100,000, O(n¬≤) will TLE (Time Limit Exceeded)
  • Signals: Shows you didn't think about optimization
  • Brute force is fine as starting point: But you must optimize from there
  • No interviewer patience: They'll stop you and say "Can you do better?"
üü¢ 7. Optimized Solution (Go)
‚úÖ Two Pointers Approach - O(n) Time, O(1) Space
Start with pointers at both ends. Move the pointer pointing to the shorter line inward. This guarantees we won't miss the optimal solution.
package main

import "fmt"

// Container With Most Water - Optimized Two Pointers
func maxArea(height []int) int {
    // Early exit: edge case
    if len(height) < 2 {
        return 0
    }
    
    // Initialize pointers at both ends
    left := 0
    right := len(height) - 1
    maxWater := 0
    
    // Converge inward
    for left < right {
        // Calculate area with current pair
        width := right - left
        h := height[left]
        if height[right] < h {
            h = height[right]  // Use min(left, right)
        }
        area := width * h
        
        // Update maximum
        if area > maxWater {
            maxWater = area
        }
        
        // Move the pointer pointing to SHORTER line
        // Why? Because moving the taller one can't increase area
        // (width decreases, height is limited by shorter line)
        if height[left] < height[right] {
            left++  // Move shorter pointer
        } else {
            right--  // Move shorter pointer
        }
    }
    
    return maxWater
}

// Helper function for minimum
func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}

func main() {
    // Example 1
    height1 := []int{1, 8, 6, 2, 5, 4, 8, 3, 7}
    fmt.Println(maxArea(height1))  // Output: 49
    
    // Example 2: Edge case - two elements
    height2 := []int{2, 3}
    fmt.Println(maxArea(height2))  // Output: 2
    
    // Example 3: All same height
    height3 := []int{5, 5, 5, 5, 5}
    fmt.Println(maxArea(height3))  // Output: 20 (any pair except first and last)
}
üí° Key Code Comments Explained:
  • "Move the pointer pointing to SHORTER line": This is the crucial insight. If we move the taller pointer inward, width decreases and height stays limited by the shorter line ‚Üí area can only decrease or stay same.
  • "Using min(left, right)": Water container height is limited by the shorter wall
  • One pass through array: We're guaranteed to find optimal because we explore all promising pairs
üß† Mathematical Proof of Correctness

Claim: We never skip the optimal solution by moving the shorter pointer.

Proof: Suppose optimal solution is (i, j) with i < j. If during our algorithm we skip this pair:

  • We moved because height[left] or height[right] was shorter
  • If we moved left pointer past i: area(i, j) ‚â§ area we found (width was smaller or equal, height limited by min)
  • If we moved right pointer past j: similar logic applies
  • Thus, we never miss the optimal by this greedy movement

‚öôÔ∏è 9. Complexity Analysis
‚úÖ Time & Space Complexity

Time Complexity: O(n)

  • Single pass through array with two pointers
  • Each element is visited at most once
  • Left pointer moves right, right pointer moves left
  • They converge: left + right ‚â§ array length iterations

Space Complexity: O(1)

  • Only using pointer variables (left, right, maxWater)
  • No extra data structures (arrays, hash maps, queues)
  • Perfect for memory-constrained environments

Compared to Brute Force:

  • Brute Force: O(n¬≤) time ‚Üí For n=100,000: 10 billion operations ‚Üí TLE
  • Two Pointers: O(n) time ‚Üí For n=100,000: 100,000 operations ‚Üí ‚úÖ Accepted
Why This Meets Software Systems's Bar:
  • Linear time: Processes data in single pass
  • Constant space: Works with huge datasets without memory overhead
  • Scalable: 100,000 elements ‚Üí 1,000,000 elements, performance barely changes
  • Production-ready: Software Systems can use this in actual microservices
üü£ 10. Software Systems Follow-Up Questions
üü£ Advanced Questions Software Systems Will Ask

Q1: "What if the input is a stream (you can't see all data upfront)?"

A: Two-pointer doesn't work on streams. You'd need a sliding window with a deque or segment tree. However, if the stream is sorted, you could maintain pointers to sliding window and recalculate max dynamically.

Q2: "What if we have memory constraints and can only store O(log n) space?"

A: Still works! O(log n) is fine for a few pointers and variables. The two-pointer approach is still O(1) space, which is better than O(log n).

Q3: "Can we parallelize this? What if data is on different servers?"

A: This specific problem is hard to parallelize because two-pointers are inherently sequential. However, if we had to: split data into chunks, solve independently with local two-pointers, then merge results. Trade-off: communication overhead vs compute speedup.

Q4: "What if the heights are floating-point numbers instead of integers?"

A: Algorithm doesn't change. Just use float type. Be careful with floating-point comparison (use epsilon for equality checks). Complexity remains O(n).

Q5: "What if we want to return the actual indices, not just the area?"

A: Store indices instead of area values. When we find new max area, save (leftIndex, rightIndex). Return pair instead of int. Complexity still O(n) time, O(1) space (ignoring return value size).

Q6: "What if we need to find the top-K containers with most water?"

A: Modify to store all (area, indices) pairs in a max-heap. After single two-pointer pass, extract top-K. Time: O(n + k log n), Space: O(n) for heap.

Pro Tip for Following Up:
  • When interviewer asks follow-up, acknowledge the constraint change
  • Say: "With [new constraint], the two-pointer approach doesn't directly apply. We'd need to..."
  • Show flexibility: many patterns can adapt or combine
  • Always mention complexity of new approach
  • If unsure, ask: "Should we optimize for time or space with this new constraint?"
