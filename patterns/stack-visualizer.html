📚 Prerequisites
  • Arrays vs Linked Lists: Understand that a stack is an Abstract Data Type (ADT). It can be implemented under the hood using either a dynamic array or a linked list.
  • Dynamic Arrays: When implemented with arrays (like in Python lists or Go slices), pushing might occasionally trigger an O(N) reallocation if the underlying memory capacity is reached.
  • Pointers: A top pointer (or array index) is tracked to keep O(1) time complexity for Push, Pop, and Peek.
💡 Pro Interview Tips
  • Watch out for Monotonic Stacks: If an interview question asks to find the "Next Greater Element" or relies on maintaining a sorted order while pushing/popping, use a Monotonic Stack pattern.
  • LIFO use-cases: Recognize stack problems when reversing items (string reversal), managing nested structures (Valid Parentheses), or tracking state (DFS, Browser History).
  • Edge Cases: ALWAYS check if the stack is empty() before calling pop() or peek() to avoid fatal Runtime Exceptions/Panics!
