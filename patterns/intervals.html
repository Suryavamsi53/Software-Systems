Intervals Pattern

Merging and scheduling overlapping ranges

üîç How to Spot This Pattern: The 3-Prong Detection System
üìä INPUT SIGNAL
  • List of ranges/intervals
  • [start, end] pairs
  • Overlapping possible
  • Need sorting
‚ùì QUESTION SIGNAL
  • Merge overlapping
  • Insert into intervals
  • Find intersection
  • Schedule meetings
‚ö° CONSTRAINT SIGNAL
  • Sort by start/end
  • Detect overlap
  • Sequential check
  • Merge when overlaps
üìã Basic Requirements to Solve Problems
‚úì Prerequisites You Need:
  • Understanding intervals: [start, end) or [start, end] notation
  • Sorting intervals by start point or custom criteria
  • Detecting overlap: when two intervals intersect
  • Computing intersection and union of ranges
üéØ Key Skills to Master:
  • Merge overlapping intervals: sort by start, merge if overlapping
  • Find intersection: check if intervals overlap and compute overlap region
  • Insert interval: insert new interval into merged list
  • Remove intervals: exclude given range or remove covered intervals
  • Interval scheduling: select maximum non-overlapping intervals
  • Handle edge cases: empty list, single interval, adjacent intervals
üí° Mental Model:

"Sort intervals by start point. Iterate through and merge overlapping ones. Current overlaps with previous if current.start <= previous.end."

‚úÖ 1. What Amazon EXPECTS
Interviewer's Checklist
  • Sorting: Do you recognize that sorting by start time is usually the first step?
  • Overlap Logic: Can you correctly identify if [a, b] and [c, d] overlap? (i.e., b >= c).
  • Edge Cases: Handling intervals that touch (e.g., [1,2] and [2,3]) vs strictly overlapping.
  • In-Place vs New List: Knowing when to merge in-place vs creating a result list.
üß† 2. Core Concept Theory
Sorting by Start Time

This pattern handles problems involving intervals that may overlap. The key insight is that after sorting by start time, two intervals [a, b] and [c, d] overlap if b >= c.

Use when:

  • Merging overlapping intervals.
  • Finding interval intersections.
  • Scheduling problems (meeting rooms).
üéØ 3. Real-World Software Systems Scenario
Meeting Room Scheduling

Scenario: You have a list of meeting time blocks requested by employees. You need to merge consecutive meetings to show "busy" blocks on a calendar, or find a free slot for a new meeting.

Merging Overlapping Intervals

üß© 5. Canonical Software Systems Problem
Merge Intervals (LeetCode 56)

Problem: Given an array of intervals where intervals[i] = [start, end], merge all overlapping intervals, and return an array of the non-overlapping intervals that cover all the intervals in the input.

Input: intervals = [[1,3],[2,6],[8,10],[15,18]]
Output: [[1,6],[8,10],[15,18]]

üî¥ 6. Brute Force Analysis
Compare All Pairs

Compare every interval with every other interval to check for overlaps.

Complexity: O(N¬≤) time. Sorting reduces this to O(N log N).

üü¢ 7. Optimized Solution
func merge(intervals [][]int) [][]int {
    if len(intervals) <= 1 {
        return intervals
    }
    
    // Sort by start time
    sort.Slice(intervals, func(i, j int) bool {
        return intervals[i][0] < intervals[j][0]
    })
    
    merged := [][]int{}
    merged = append(merged, intervals[0])
    
    for i := 1; i < len(intervals); i++ {
        current := intervals[i]
        last := merged[len(merged)-1]
        
        if current[0] <= last[1] {
            // Overlap: extend end time
            if current[1] > last[1] {
                last[1] = current[1]
            }
        } else {
            // No overlap: add new interval
            merged = append(merged, current)
        }
    }
    
    return merged
}

class Solution {
    public int[][] merge(int[][] intervals) {
        Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));
        
        LinkedList merged = new LinkedList<>();
        for (int[] interval : intervals) {
            if (merged.isEmpty() || merged.getLast()[1] < interval[0]) {
                merged.add(interval);
            } else {
                merged.getLast()[1] = Math.max(merged.getLast()[1], interval[1]);
            }
        }
        
        return merged.toArray(new int[merged.size()][]);
    }
}

class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        intervals.sort(key=lambda x: x[0])
        merged = []
        
        for interval in intervals:
            if not merged or merged[-1][1] < interval[0]:
                merged.append(interval)
            else:
                merged[-1][1] = max(merged[-1][1], interval[1])
                
        return merged

‚öôÔ∏è 9. Complexity Analysis
Time & Space

Time Complexity: O(N log N)

Dominated by the sorting step. The merge pass is O(N).

Space Complexity: O(N)

To store the merged intervals (or O(log N) for sorting stack).

üü£ 10. Software Systems Follow-up Questions
Variations

Q: What if intervals are already sorted?
A: Then it's O(N) time.

Q: How to handle a stream of intervals?
A: Use a BST (TreeMap/TreeSet) to store intervals. Insert takes O(log N) and can merge neighbors.
