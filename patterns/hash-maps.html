Interviewer's Checklist

üß† 2. Core Concept Theory
Instant Access

Hash Maps allow you to store key-value pairs and retrieve them in constant time O(1) on average.

Use when:

  • You need to check for existence of an element.
  • You need to count frequencies of elements.
  • You need to map one value to another (e.g., ID to User).
üéØ 3. Real-World Software Systems Scenario
Scenario 1: Product Deduplication & Counting

Context: Top Tech processes billions of clickstream events. We need to aggregate clicks per product (ASIN) to calculate "Most Popular" lists in real-time.

Solution: Use a Hash Map (or distributed Key-Value store like Redis/DynamoDB) where Key = ASIN and Value = ClickCount. This allows O(1) updates.

Scenario 2: Shopping Cart (Session Storage)

Context: When a user adds items to their cart, we need to store this temporary state linked to their UserID.

Solution: A Hash Map allows instant retrieval of the user's cart using their UserID as the key. In a distributed system, this is often sharded (Consistent Hashing).

Problem: Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target.

Input: nums = [2,7,11,15], target = 9
Output: [0,1]

Nested Loops

Check every pair (i, j).

Time Complexity: O(n¬≤)

Why Software Systems Rejects: O(n¬≤) is unacceptable for large datasets. Hash Map reduces this to O(n).

üü¢ 7. Optimized Solution
func twoSum(nums []int, target int) []int {
    // Map to store number -> index
    seen := make(map[int]int)

    for i, num := range nums {
        complement := target - num
        
        // Check if complement exists in map
        if idx, ok := seen[complement]; ok {
            return []int{idx, i}
        }
        
        // Store current number and index
        seen[num] = i
    }
    return nil
}

class Solution {
    public int[] twoSum(int[] nums, int target) {
        // Map to store number -> index
        Map map = new HashMap<>();

        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i];
            
            if (map.containsKey(complement)) {
                return new int[] { map.get(complement), i };
            }
            
            map.put(nums[i], i);
        }
        
        throw new IllegalArgumentException("No two sum solution");
    }
}

‚öôÔ∏è 9. Complexity Analysis
Time & Space

Time Complexity: O(n)

We traverse the list containing n elements exactly once. Each lookup in the table costs only O(1) time.

Space Complexity: O(n)

The extra space required depends on the number of items stored in the hash table, which stores at most n elements.

üü£ 10. Software Systems Follow-up Questions
Variations

Q: What if the array is sorted?
A: Use Two Pointers! It takes O(n) time but only O(1) space, which is better than Hash Map's O(n) space.

Q: What if we need to return all unique pairs?
A: You need to handle duplicates carefully, perhaps by sorting first or using a Set to store result pairs.

üèóÔ∏è 11. System Design Connection
Distributed Hash Tables (DHT) & Consistent Hashing

In System Design interviews (SDE-2+), Hash Maps scale horizontally via Sharding. Instead of hash(key) % array_size, we use Consistent Hashing to map keys to servers. This minimizes data movement when servers are added or removed (scaling).

Top Tech DynamoDB is essentially a massive, distributed, persistent Hash Map.

üéì 12. Knowledge Check
Q1: Why is the worst-case time complexity of a Hash Map O(n)? ‚ñº

If the hash function is poor, all keys might map to the same bucket (collision). If the collision resolution uses a linked list (chaining), searching becomes a linear scan of that list, which is O(n).

Q2: How does a Hash Map handle resizing? ‚ñº

When the load factor (elements / buckets) exceeds a threshold (e.g., 0.75), the map creates a new, larger array (usually double size) and re-hashes all existing elements into the new buckets. This is an O(n) operation but happens rarely (amortized O(1)).

Q3: Can we use a mutable object (like a List) as a Key? ‚ñº

Generally, no. If the object changes after being inserted, its hash code changes. You won't be able to find it in the correct bucket anymore. In Python, lists cannot be keys; tuples can. In Java, you can but it's dangerous.
