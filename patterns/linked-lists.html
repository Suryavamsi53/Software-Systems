Interviewer Checklist
  • Pointer Hygiene: Can you handle next pointers without losing reference to the rest of the list?
  • Edge Cases: Do you handle empty lists, single-node lists, and cycles correctly?
  • Dummy Head: Do you use a dummy head node to simplify edge cases (like deleting the head)?
  • In-Place Operations: Can you reverse or manipulate the list with O(1) space?
ðŸ§  2. Core Concept Theory
Nodes and Pointers

A Linked List is a linear data structure where elements are not stored in contiguous memory. Each element (node) points to the next.

Key Techniques:

  • Dummy Head: A placeholder node before the actual head to handle edge cases where the head changes.
  • Two Pointers: Using 'prev', 'curr', and 'next' pointers to manipulate links.
  • Fast & Slow: For cycle detection and finding the middle (covered in specialized pattern).
ðŸ§© 5. Canonical Software Systems Problem
Reverse Linked List (LeetCode 206)

Problem: Given the head of a singly linked list, reverse the list, and return the reversed list.

Input: head = [1,2,3,4,5]
Output: [5,4,3,2,1]

Why Software Systems Loves It: It tests raw pointer manipulation skills. It's often a sub-step in harder problems (like Palindrome Linked List or Add Two Numbers).

/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func reverseList(head *ListNode) *ListNode {
    var prev *ListNode = nil
    curr := head
    
    for curr != nil {
        // 1. Save next node
        nextTemp := curr.Next
        
        // 2. Reverse pointer
        curr.Next = prev
        
        // 3. Move prev and curr forward
        prev = curr
        curr = nextTemp
    }
    
    return prev // prev is the new head
}

class Solution {
    public ListNode reverseList(ListNode head) {
        ListNode prev = null;
        ListNode curr = head;
        
        while (curr != null) {
            ListNode nextTemp = curr.next;
            curr.next = prev;
            prev = curr;
            curr = nextTemp;
        }
        
        return prev;
    }
}

class Solution:
    def reverseList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        prev = None
        curr = head
        
        while curr:
            next_temp = curr.next
            curr.next = prev
            prev = curr
            curr = next_temp
            
        return prev
