üß† 2. Core Concept Theory
Order Brings Efficiency

Sorting transforms unstructured data into structured data. Once sorted, many problems become O(N) or O(log N) that were otherwise O(N¬≤).

Key Insight: If the problem involves "intervals", "overlapping", "closest", or "k-th", sorting is likely the first step.

üéØ 3. Real-World Software Systems Scenario
Order Fulfillment Scheduling

Scenario: Software Systems has thousands of delivery trucks with specific availability windows. To maximize efficiency, we need to merge overlapping availability slots to see continuous coverage blocks.

Solution: Sort all availability intervals by start time. Iterate once to merge overlaps. This is exactly the "Merge Intervals" pattern.

üß© 5. Canonical Software Systems Problem
Merge Intervals (LeetCode 56)

Problem: Given an array of intervals where intervals[i] = [start, end], merge all overlapping intervals, and return an array of the non-overlapping intervals that cover all the intervals in the input.

Input: intervals = [[1,3],[2,6],[8,10],[15,18]]
Output: [[1,6],[8,10],[15,18]]

üî¥ 6. Brute Force Analysis
Compare All Pairs

For every interval, compare it with every other interval to check for overlap. If they overlap, merge them and repeat.

Time Complexity: O(N¬≤) or worse depending on implementation.

Why it fails: Inefficient. Sorting allows us to check only adjacent intervals.

üü¢ 7. Optimized Solution
import (
    "sort"
)

func merge(intervals [][]int) [][]int {
    if len(intervals) <= 1 {
        return intervals
    }
    
    // 1. Sort by start time
    sort.Slice(intervals, func(i, j int) bool {
        return intervals[i][0] < intervals[j][0]
    })
    
    result := [][]int{}
    current := intervals[0]
    result = append(result, current)
    
    for i := 1; i < len(intervals); i++ {
        nextInterval := intervals[i]
        
        // 2. Check overlap
        // current.end >= next.start
        if result[len(result)-1][1] >= nextInterval[0] {
            // Merge: update end time to max of both
            if nextInterval[1] > result[len(result)-1][1] {
                result[len(result)-1][1] = nextInterval[1]
            }
        } else {
            // No overlap, add next interval
            result = append(result, nextInterval)
        }
    }
    
    return result
}

class Solution {
    public int[][] merge(int[][] intervals) {
        if (intervals.length <= 1) return intervals;
        
        // Sort by start time
        Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));
        
        List result = new ArrayList<>();
        int[] current = intervals[0];
        result.add(current);
        
        for (int[] interval : intervals) {
            if (current[1] >= interval[0]) {
                // Overlap: merge
                current[1] = Math.max(current[1], interval[1]);
            } else {
                // No overlap: move to next
                current = interval;
                result.add(current);
            }
        }
        
        return result.toArray(new int[result.size()][]);
    }
}

class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        intervals.sort(key=lambda x: x[0])
        merged = []
        
        for interval in intervals:
            # if list is empty or no overlap
            if not merged or merged[-1][1] < interval[0]:
                merged.append(interval)
            else:
                # overlap, merge
                merged[-1][1] = max(merged[-1][1], interval[1])
        
        return merged

‚öôÔ∏è 9. Complexity Analysis
Time & Space

Time Complexity: O(N log N)

Sorting takes O(N log N). The linear scan takes O(N). Total is dominated by sorting.

Space Complexity: O(log N) or O(N)

Depends on the sorting implementation (stack space). If we count the output array, it's O(N).

üü£ 10. Software Systems Follow-up Questions
Variations

Q: What if the intervals are already sorted?
A: Then the complexity is O(N) time.

Q: How to handle a stream of intervals?
A: Use a BST (TreeMap in Java) to store intervals. Adding an interval takes O(log N) and you can check/merge neighbors efficiently.
