Google Interview Questions

Top 50 most frequent questions for Google SDE interviews, categorized by topic.

üöÄ What Google Looks For

Google interviews are known for ambiguity and scale. Unlike Amazon, they focus less on Leadership Principles and more on "Googlyness" (navigating ambiguity, valuing feedback, challenging status quo).

  • Algorithms: Heavy focus on Graphs, DP, and Trees.
  • Code Quality: Speed and correctness are paramount.
  • Clarification: You MUST ask clarifying questions to define the vague problem.
1. Graphs & Trees (High Priority)
  • 1. Word Ladder IIHard
    Evaluates: BFS, Backtracking, Optimization.
    Find all shortest transformation sequences from startWord to endWord. Requires BFS to find distance and DFS/Backtracking to reconstruct paths.
  • 2. Course Schedule IIMedium
    Evaluates: Topological Sort, Cycle Detection.
    Return the ordering of courses to finish all. Use Kahn's Algorithm (BFS) or DFS with 3-state coloring (White/Gray/Black) to detect cycles.
  • 3. Number of Islands IIHard
    Evaluates: Union-Find (Disjoint Set).
    Dynamically add land and count islands. Use Union-Find to merge components in O(1) (amortized) time per operation.
  • 4. Diameter of Binary TreeEasy
    Evaluates: Tree DFS, Recursion.
    Find the longest path between any two nodes. Compute depth of left and right subtrees at each node; update max diameter with left_depth + right_depth.
  • 5. Alien DictionaryHard
    Evaluates: Topological Sort, Edge Cases.
    Derive character order from a sorted list of alien words. Build a graph where char1 -> char2 if char1 comes before char2. Perform Topo Sort.
  • 6. Lowest Common Ancestor (Binary Tree)Medium
    Evaluates: Tree Traversal, Recursion.
    Find LCA of two nodes. If root matches either p or q, return root. Else recurse left and right. If both return non-null, root is LCA.
  • 7. Cracking the SafeHard
    Evaluates: Euler Path, Hierholzer's Algorithm.
    Find shortest string containing all n-length codes. Model as a De Bruijn sequence graph problem.
2. Dynamic Programming
  • 8. Split Array Largest SumHard
    Evaluates: Binary Search on Answer + Greedy.
    Split array into m subarrays to minimize the largest sum. Binary search the answer range [max(nums), sum(nums)].
  • 9. Longest Increasing Path in a MatrixHard
    Evaluates: DFS + Memoization.
    Find the longest path in a matrix where values increase. Use DFS with memoization dp[i][j] to store max path starting at (i, j).
  • 10. Edit DistanceMedium
    Evaluates: 2D DP.
    Min operations to convert word1 to word2. dp[i][j] depends on insert, delete, or replace costs.
  • 11. Maximum Subarray (Kadane's)Easy
    Evaluates: Array Optimization.
    Find contiguous subarray with largest sum. current_max = max(num, current_max + num).
3. Arrays & Strings
  • 12. Next PermutationMedium
    Evaluates: Mathematical Logic.
    Find next lexicographical permutation. 1. Find first decreasing element from right. 2. Swap with next larger. 3. Reverse suffix.
  • 13. 3SumMedium
    Evaluates: Two Pointers.
    Find unique triplets summing to zero. Sort array, iterate i, then use two pointers for j and k.
  • 14. Text JustificationHard
    Evaluates: String Manipulation, Edge Cases.
    Pack words into lines of width `maxWidth`. Distribute spaces evenly. Handle last line (left-justified) differently.
  • 15. Meeting Rooms IIMedium
    Evaluates: Heap / Sorting.
    Min conference rooms required. Sort by start time. Use Min-Heap to track end times of active meetings.
4. System Design (Google Scale)
  • 16. Design Google Drive / DropboxSystem Design
    Evaluates: File Storage, Synchronization, Consistency.
    Focus on chunking files (block storage), deduplication, metadata DB (ACID), and synchronization service (long-polling/websockets).
  • 17. Design YouTubeSystem Design
    Evaluates: CDN, Blob Storage, Transcoding.
    Video upload -> Queue -> Transcoding Service (different formats) -> S3/Blob Store -> CDN. Metadata in SQL/NoSQL.
  • 18. Design a Web CrawlerSystem Design
    Evaluates: Distributed Systems, BFS, Politeness.
    URL Frontier (Queue), DNS Resolver, Content Downloader, Deduplication (Bloom Filter), Storage. Handle cycles and robots.txt.
  • 19. Design Google Search TypeaheadSystem Design
    Evaluates: Tries, Caching, Latency.
    Use a Trie to store prefixes. Store top 5 hot searches at each node. Cache heavily.
5. Googlyness (Behavioral)

Google values "Googlyness": navigating ambiguity, bias for action, collaboration, and humility.

  • 20. Tell me about a time you navigated ambiguity.
    Evaluates: Independence, Problem Solving.
    Focus on how you gathered requirements, made assumptions, validated them, and delivered despite unclear instructions.
  • 21. Tell me about a time you challenged the status quo.
    Evaluates: Innovation, Courage.
    Describe a situation where you improved a process or technology, even if it wasn't broken, to gain efficiency or quality.
  • 22. How do you handle constructive criticism?
    Evaluates: Humility, Growth Mindset.
    Show that you value feedback as a tool for growth, not a personal attack. Give a specific example of changing behavior based on feedback.
  • 23. Describe a time you failed.
    Evaluates: Resilience, Learning.
    Don't use a fake failure. Pick a real one, own it, explain what you learned, and how you prevent it now.
