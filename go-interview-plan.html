Classic Interview Question

Q: Channel vs Mutex?

A: Use Channels for passing data/communication between goroutines. Use Mutex for protecting shared state (caches, counters) from race conditions.

6Ô∏è‚É£ Channels (Must Know)

  • Closing: Sender should close. Closing a closed channel panics.
  • Reading Closed: Returns zero value + false immediately.

üß† Part 4: Memory & Performance
Interview-Safe GC Explanation:

"Go uses a concurrent, tri-color mark-and-sweep Garbage Collector optimized for low pause times (latency) rather than high throughput."

Escape Analysis: The compiler decides whether to allocate on Stack (fast) or Heap (GC managed). If a reference returns from a function, it "escapes" to the Heap.

Day 1‚Äì3: The Foundation (Core Syntax & Data Structures) ‚ñº

Concepts Briefly

  • Slices: Dynamic arrays. Think of them as a window into an underlying array. They grow automatically.
  • Maps: Key-value storage (Hash Table). Fast lookups O(1). Unordered.
  • Structs: Custom data types grouping related fields. No classes, just structs.
  • Pointers: Direct memory addresses. Used to share data without copying.

Real-World Scenarios

  • Slices: Storing a list of active WebSocket connections in a chat room.
  • Maps: An in-memory cache storing userID -> UserProfile for fast API responses.
  • Structs: Defining the Order payload for a JSON REST API.

Ugly vs Clean Code

// ‚ùå Ugly: Returning pointer to slice (unnecessary)
func getItems() *[]int {
    s := []int{1, 2}
    return &s
}

// ‚úÖ Clean: Return slice directly (it's lightweight)
func getItems() []int {
    return []int{1, 2}
}

Interface Implementation

Go interfaces are implemented implicitly. No implements keyword.

type Shape interface {
    Area() float64
}

type Rect struct { w, h float64 }

// Implicitly implements Shape
func (r Rect) Area() float64 { return r.w * r.h }
Day 4‚Äì6: Concurrency Mastery (Goroutines & Channels) ‚ñº

Concepts Briefly

  • Goroutines: Functions running concurrently. Extremely cheap (2KB stack).
  • Channels: Pipes for goroutines to talk. "Don't communicate by sharing memory, share memory by communicating."
  • WaitGroup: Waits for a collection of goroutines to finish.
  • Mutex: Locks data so only one goroutine accesses it at a time.

Real-World Scenarios

  • Goroutines: Handling 10,000 concurrent HTTP requests on a web server.
  • Channels: A job queue where an API handler sends tasks to background workers (e.g., email sending).
  • Mutex: Updating a global "request counter" safely across multiple threads.

Ugly vs Clean Code

// ‚ùå Ugly: Sleep to wait for goroutine
go doWork()
time.Sleep(1 * time.Second)

// ‚úÖ Clean: Use WaitGroup
var wg sync.WaitGroup
wg.Add(1)
go func() {
    defer wg.Done()
    doWork()
}()
wg.Wait()

Select with Timeout Pattern

Essential for preventing goroutine leaks in production.

select {
case res := <-ch:
    fmt.Println("Received:", res)
case <-time.After(2 * time.Second):
    fmt.Println("Timeout! Operation took too long.")
}
Day 7‚Äì9: DSA Patterns in Go ‚ñº

Concepts Briefly

  • Two Pointers: Using two indices to traverse a sequence (often sorted) to find pairs or reverse.
  • Sliding Window: Maintaining a subset of data that moves across the input (e.g., subarray sum).
  • Hash Maps: Using a map to track frequency or existence for O(1) checks.

Real-World Scenarios

  • Two Pointers: Checking if a user's activity log is a palindrome (same sequence start/end).
  • Sliding Window: Rate limiting (allowing only 5 requests per 1-minute window).
  • Hash Maps: Finding duplicate transactions in a payment stream.

Go DSA Idioms

// Stack (LIFO) using Slice
stack := []int{}
stack = append(stack, 10)    // Push
val := stack[len(stack)-1]   // Peek
stack = stack[:len(stack)-1] // Pop

// Queue (FIFO) using Slice
queue := []int{}
queue = append(queue, 10) // Enqueue
val := queue[0]           // Peek
queue = queue[1:]         // Dequeue

Custom Sorting

type User struct { Name string; Age int }
users := []User{{"Bob", 30}, {"Alice", 25}}

// Sort by Age using sort.Slice
sort.Slice(users, func(i, j int) bool {
    return users[i].Age < users[j].Age
})
Day 10‚Äì11: Backend Development (REST, JSON, Context) ‚ñº

Concepts Briefly

  • REST API: Standard way to build web services using HTTP methods (GET, POST).
  • Context: Carries deadlines, cancellation signals, and request-scoped values across API boundaries.
  • JSON: Standard data format. Go uses struct tags (`json:"name"`) to map fields.

Real-World Scenarios

  • Context: A user cancels a search request on the frontend; the backend cancels the heavy DB query immediately to save resources.
  • Middleware: Authentication middleware that checks a JWT token before allowing access to the handler.

Ugly vs Clean Code

// ‚ùå Ugly: Ignoring context
func handler(w http.ResponseWriter, r *http.Request) {
    longProcess() // Keeps running even if client disconnects
}

// ‚úÖ Clean: Respecting context
func handler(w http.ResponseWriter, r *http.Request) {
    ctx := r.Context()
    select {
    case <-time.After(2 * time.Second):
        fmt.Fprintln(w, "Done")
    case <-ctx.Done():
        fmt.Println("Request cancelled")
    }
}

Middleware Pattern

func LoggingMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        next.ServeHTTP(w, r)
        fmt.Printf("%s %s took %v\n", r.Method, r.URL.Path, time.Since(start))
    })
}

Classic Gotcha: Loop Variables

Before Go 1.22, loop variables were shared per iteration.

// ‚ùå Wrong (prints same value)
for _, v := range values {
    go func() { fmt.Println(v) }()
}

// ‚úÖ Correct (pass as arg)
for _, v := range values {
    go func(val int) { fmt.Println(val) }(v)
}
