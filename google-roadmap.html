Google SDE Roadmap

A specialized 10-week path to crack Google interviews (L3/L4/L5)

‚≠ê Google Strategy

Google interviews prioritize Graph Algorithms, Dynamic Programming, and Ambiguity Resolution. Unlike Amazon, code speed and algorithmic optimality are paramount.

Phase 1: Graph Mastery (High Priority) Weeks 1-3

Google loves graphs. You must be able to implement BFS/DFS, Union-Find, and Dijkstra blindfolded.

Graph Traversals Live BFS, DFS, Connected Components
Union-Find Live Disjoint Sets, Cycle Detection
Topological Sort Live Dependency Resolution, Course Schedule
Shortest Path & MST Live Dijkstra, Prim's, Kruskal's
Phase 2: Dynamic Programming & Recursion Weeks 4-6

Expect 2D DP, Memoization, and Hard variants of standard problems.

DP Patterns Live Knapsack, LCS, LIS, Grid Paths
Backtracking Live Permutations, Subsets, N-Queens
String DP Live Edit Distance, Wildcard Matching
Phase 3: Advanced Data Structures Weeks 7-8

Tries, Segment Trees, and Heaps are fair game.

Tries (Prefix Trees) Live Autocomplete, Word Search II
Heaps & Priority Queues Live Merge K Lists, Median Stream
Intervals Live Merge Intervals, Meeting Rooms II
Phase 4: System Design & Googlyness Weeks 9-10

For L4+, System Design is critical. For all levels, behavioral questions matter.

System Design Live Scalability, Load Balancing, Caching
Google Questions Live Top 50 Google-specific problems
