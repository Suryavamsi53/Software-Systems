üéØ Amazon Interview Approach Guide

Master the systematic approach to solve any coding problem during your interview

üìã The 5-Step Amazon Interview Framework
Step 1: Clarify the Problem (2-3 minutes)
  • Read carefully: Don't rush. Re-read the problem statement.
  • Ask clarifying questions:
    • What are the input constraints? (size, range, duplicates?)
    • Are there edge cases I should handle? (empty, single element, negative numbers?)
    • What is the expected output format?
    • Any time/space constraints mentioned?
  • Examples: Work through 2-3 examples with the interviewer
  • Confirm understanding: "So the problem is asking me to..."
Step 2: Identify the Pattern (2-3 minutes)
  • Use the 3-Prong Detection:
    • What INPUT SIGNAL do I see?
    • What QUESTION SIGNAL is being asked?
    • What CONSTRAINT SIGNAL is given?
  • Pattern matching: "This looks like a [Pattern Name] problem because..."
  • Think aloud: Help interviewer follow your reasoning
  • Multiple approaches? Acknowledge if multiple patterns could apply
Step 3: Plan Your Approach (3-5 minutes)
  • Discuss before coding: Never jump straight to code
  • Explain your strategy:
    • What data structure will you use and why?
    • What is the main algorithm/approach?
    • Walk through the algorithm on one of your examples
  • Estimate complexity: "This approach will be O(n time) and O(n space)"
  • Get approval: "Does this approach sound good to you?" before coding
  • Consider optimizations: "We could optimize this further by..."
Step 4: Implement the Solution (8-12 minutes)
  • Code structure:
    • Write function signature first
    • Handle edge cases at the start
    • Use clear variable names (not just i, j, x)
    • Add comments for non-obvious steps
  • Narrate while coding: Explain what each section does
  • Test as you go: "In this section, I initialize the result array"
  • Don't write too fast: Quality over speed
  • Be organized: Format code properly, use indentation
Step 5: Test & Optimize (3-5 minutes)
  • Walk through examples: Trace through your code with the examples
  • Edge cases: Test empty input, single element, all duplicates, etc.
  • Check for bugs: Off-by-one errors, null pointer issues, infinite loops
  • Complexity analysis: Count operations, analyze loops
  • Optimization discussion: Can we do better? Trade-offs?
  • Code cleanup: Remove debug code, improve readability

‚è±Ô∏è How to Explain Time & Space Complexity in an Interview
Time Complexity Explanation

Don't just say "O(n log n)" ‚Äî explain it:

‚ùå "The time complexity is O(n log n)"

‚úÖ "The time complexity is O(n log n) because:
  - We need to sort the input [outer n operations]
  - Sorting takes O(n log n) time
  - Then we iterate through [O(n) more operations]
  - Overall: O(n log n) dominates, so O(n log n)"
Space Complexity Explanation

Be specific about what uses space:

‚ùå "The space complexity is O(n)"

‚úÖ "The space complexity is O(n) because:
  - We use a hash map to store elements: O(n)
  - Result array: O(n)
  - Input doesn't count (given)
  - Total: O(n) for the hash map"
Discussing Trade-offs

Show you understand optimization:

"My current approach is O(n log n) time and O(n) space.
We could optimize the time to O(n) using a hash map, but
that would increase space to O(n). In this case, O(n) time
is worth the extra space, so that's my preferred solution."

üìù Example: How to Solve "Two Sum" in an Interview
üìå Problem Statement:

Given an array of integers nums and an integer target, return the indices of the two numbers that add up to the target. You may not use the same element twice. You can return the answer in any order.

üí¨ Step 1: Clarify (What you should SAY)

"Let me clarify the problem:
‚Ä¢ I need to find two numbers that add to the target
‚Ä¢ Return their indices (not values)
‚Ä¢ Can't use the same element twice
‚Ä¢ Is the array sorted? [Wait for answer]
‚Ä¢ Are there guaranteed to be two numbers that sum to target?

Let me trace through an example:
nums = [2, 7, 11, 15], target = 9
Output: [0, 1] because 2 + 7 = 9"

üîç Step 2: Identify Pattern (What you should SAY)

"This looks like a Hash Maps pattern because:
‚Ä¢ INPUT: Array of integers
‚Ä¢ QUESTION: Find two numbers with specific sum
‚Ä¢ CONSTRAINT: Need O(n) time, not O(n¬≤)

I can use the complementary sum approach:
For each number, check if (target - number) exists in my seen numbers"

üìã Step 3: Plan (What you should SAY)

"My approach:
1. Create a hash map to store value ‚Üí index
2. Iterate through the array
3. For each number, calculate complement = target - num
4. Check if complement exists in hash map
5. If yes, return [complement_index, current_index]
6. If no, add current number to hash map

Time: O(n) single pass
Space: O(n) for hash map
Does this approach look good to you?"

üíª Step 4: Code (Implementation) 
def twoSum(nums, target):
    """
    Find two indices whose values sum to target.
    
    Args:
        nums: List of integers
        target: Target sum
    
    Returns:
        List of two indices [i, j] where nums[i] + nums[j] = target
    """
    # Edge case: array too small
    if not nums or len(nums) < 2:
        return []
    
    # Hash map: value ‚Üí index
    seen = {}
    
    # Single pass through array
    for i, num in enumerate(nums):
        complement = target - num
        
        # Check if complement exists
        if complement in seen:
            return [seen[complement], i]
        
        # Add current number to map
        seen[num] = i
    
    # No solution found
    return []
‚úÖ Step 5: Test (What you should SAY)

"Let me trace through the example:
nums = [2, 7, 11, 15], target = 9

i=0, num=2: complement=7, seen={}, add 2‚Üí0
i=1, num=7: complement=2, 2 in seen! Return [0, 1] ‚úì

Edge cases:
‚Ä¢ Empty array: handled ‚úì
‚Ä¢ Single element: handled ‚úì
‚Ä¢ No solution: returns [] ‚úì

Complexity:
‚Ä¢ Time: O(n) - one pass
‚Ä¢ Space: O(n) - hash map size"
