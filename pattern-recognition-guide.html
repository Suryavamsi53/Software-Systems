Master the 3-Prong Detection System: Learn to identify the right pattern in any interview question

üîç The 3-Prong Pattern Detection System

Every interview question contains three hidden signals. Learn to recognize them, and you'll instantly know which pattern to use.

üìä INPUT SIGNAL

What data structure is involved?

  • Array or String
  • Tree or Graph
  • Linked List
  • Stream of Data
  • Matrix/2D Grid
  • Sorted vs Unsorted
‚ùì QUESTION SIGNAL

What exactly are you asked to find?

  • Find/Search a value
  • Count occurrences
  • Max/Min value
  • Rebuild/Rearrange
  • Valid path/sequence
  • Optimize subproblem
‚ö° CONSTRAINT SIGNAL

What's the limitation/requirement?

  • Time: O(n) or O(n log n)
  • Space: O(1) or O(n)
  • In-place modification
  • No sorting allowed
  • Single pass through data
  • Memory limited

‚ö° Quick Reference: When to Use Each Pattern
Two Pointers
  • Sorted array + find pair
  • Reverse array in-place
  • Remove duplicates
  • Container with most water
Sliding Window
  • Longest/shortest substring
  • Subarray with sum K
  • Max product of window
  • Anagram matching
Binary Search
  • Search in sorted array
  • Find boundary (first/last)
  • Search in rotated array
  • Peak element finding
Hash Maps/Sets
  • Two Sum problem
  • Frequency counting
  • Duplicate detection
  • Anagram grouping
Linked Lists
  • Cycle detection
  • Reverse list
  • Merge lists
  • Remove Nth node
Dynamic Programming
  • Overlapping subproblems
  • Optimal substructure
  • Longest subsequence
  • Coin change problems
‚úÖ Before You Code: The Final Action Box

Ask yourself these questions BEFORE touching the keyboard:

1. What is the input signature?
  • Array? String? Tree? Graph? Stream?
  • Sorted or unsorted?
  • Can have duplicates?
2. What exactly is being optimized?
  • Time complexity?
  • Space complexity?
  • Code simplicity?
  • In-place modification?
3. What is the constraint telling me?
  • Time limit rules out brute force?
  • Space limit rules out extra data structures?
  • Single pass requirement?
  • This suggests the pattern ‚Üí ‚úÖ
‚ö†Ô∏è Why Patterns Fail: The Interview Reality Box
‚ùå Wrong Pattern Choice

You misread the constraints. Re-check: is the array sorted? Is the window fixed or dynamic? Can you modify in-place?

‚ùå Ignoring Edge Cases

Empty array? Single element? All duplicates? Negative numbers? Think through these before coding.

‚ùå Complexity Not Optimized

You chose the right pattern but implemented it wrong. Two Pointers should be O(n), not O(n¬≤).

‚ùå Off-by-One Errors

Boundary conditions in loops, array indexing, and pointer movement are critical. Test with small inputs first.

üéì Practice Strategy
  1. Identify the Pattern (30 seconds)

    Use the 3-prong system. Input? Question? Constraint? ‚Üí Which pattern?

  2. Recall the Template (1 minute)

    Remember the core approach: two pointers setup, window boundaries, binary search logic, etc.

  3. Implement Carefully (5-10 minutes)

    Write clean code, test edge cases, optimize for the stated constraint.

  4. Optimize if Needed (2 minutes)

    Can you improve time/space? Is there a variation you haven't considered?
