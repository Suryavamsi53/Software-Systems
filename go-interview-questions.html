Go (Golang) Interview Questions: Top 200

The definitive list of 200 interview questions used by FAANG and top tech companies. Categorized by topic with concise answers.

üöÄ Interviewer's Guide

This list covers 20 categories ranging from syntax basics to advanced concurrency and system design. Each question includes a difficulty tag and what the interviewer is evaluating.

1. Go Basics & Syntax
  • 1. What is the difference between var and :=?Beginner
    Evaluates: Understanding of variable scopes and initialization.
    var allows declaring variables without initialization (zero-valued) and works at package level. := is short declaration, infers type, requires initialization, and only works inside functions.
  • 2. What are the zero values for standard types?Beginner
    Evaluates: Basic language knowledge.
    int=0, float=0.0, bool=false, string="" (empty), pointer/slice/map/chan/func/interface=nil.
  • 3. How does the init() function work?Intermediate
    Evaluates: Understanding of package initialization order.
    It runs before main(). A package can have multiple init() functions. They execute in the order imports are processed, then variable declarations, then init().
  • 4. What is the blank identifier _?Beginner
    Evaluates: Handling unused variables and imports.
    It's a write-only placeholder for values you want to discard (e.g., ignoring an error return or importing a package solely for its side effects).
  • 5. Does Go support inheritance?Intermediate
    Evaluates: OOP concepts in Go.
    No. Go uses composition via struct embedding. It achieves polymorphism through interfaces, not class hierarchy.
  • 6. What is variable shadowing?Intermediate
    Evaluates: Debugging skills and scope awareness.
    Declaring a variable in an inner scope with the same name as one in an outer scope, making the outer one inaccessible. Common with err in if blocks.
  • 7. Can you declare a variable without using it?Beginner
    Evaluates: Compiler strictness knowledge.
    No, the compiler will throw an error for unused local variables. Unused package-level variables are allowed.
  • 8. What is the difference between const and var?Beginner
    Evaluates: Immutability and compile-time evaluation.
    const is evaluated at compile-time and cannot change. var can be changed and is evaluated at runtime.
  • 9. Does Go have exceptions?Beginner
    Evaluates: Error handling philosophy.
    No. Go uses explicit error return values. It has panic and recover for truly exceptional conditions, but they shouldn't be used for control flow.
  • 10. What is the scope of a package-level variable?Beginner
    Evaluates: Visibility rules.
    It is visible throughout the entire package. If capitalized, it is exported and visible to other packages.
2. Data Types & Variables
  • 11. What is iota?Intermediate
    Evaluates: Enum creation and constant management.
    It's a predeclared identifier for creating enumerated constants. It resets to 0 at the start of a const block and increments by 1 for each line.
  • 12. Difference between byte and rune?Intermediate
    Evaluates: String handling and Unicode knowledge.
    byte is an alias for uint8 (ASCII). rune is an alias for int32 (Unicode code point).
  • 13. Are strings mutable?Beginner
    Evaluates: Memory model knowledge.
    No. Strings are immutable byte slices. To modify, you must convert to []byte or []rune, creating a copy.
  • 14. What is the size of int?Intermediate
    Evaluates: Platform dependence awareness.
    It depends on the architecture. 32-bit on 32-bit systems, 64-bit on 64-bit systems. Use int64 for fixed size.
  • 15. How do you check the type of a variable at runtime?Intermediate
    Evaluates: Reflection and type assertions.
    Use type switches switch v.(type) or the reflect package (reflect.TypeOf).
  • 16. Can you compare two structs?Intermediate
    Evaluates: Equality rules.
    Yes, if all their fields are comparable. Structs with slices or maps are not comparable using ==.
  • 17. What is a raw string literal?Beginner
    Evaluates: Syntax knowledge.
    Strings enclosed in backticks `...`. They span multiple lines and ignore escape sequences.
  • 18. How does type conversion work?Beginner
    Evaluates: Type safety.
    Go requires explicit conversion: T(v). There is no implicit casting between types (e.g., int to float).
  • 19. What is nil?Intermediate
    Evaluates: Zero value concepts.
    It is the zero value for pointers, interfaces, maps, slices, channels, and function types. It is NOT a keyword for "null" in the traditional sense.
  • 20. Can you define methods on basic types?Intermediate
    Evaluates: Type system flexibility.
    Yes, if you define a type alias (e.g., type MyInt int), you can attach methods to MyInt.
3. Arrays, Slices, Maps
  • 21. Array vs Slice?Beginner
    Evaluates: Core data structure knowledge.
    Arrays are fixed-size value types. Slices are dynamic references to arrays (pointer, len, cap).
  • 22. How does append work?Intermediate
    Evaluates: Memory allocation mechanics.
    If capacity allows, it extends length. If not, it allocates a new larger array, copies elements, and returns a new slice header.
  • 23. Difference between nil slice and empty slice?Intermediate
    Evaluates: Initialization nuances.
    nil slice has no underlying array. Empty slice []int{} has a pointer to a zero-sized array. JSON marshals them as null vs [].
  • 24. Are maps thread-safe?Intermediate
    Evaluates: Concurrency safety.
    No. Concurrent read/write panics. Use sync.RWMutex or sync.Map.
  • 25. Can you take the address of a map element?Advanced
    Evaluates: Internal implementation knowledge.
    No. Map elements can move during growth/rebalancing, invalidating pointers.
  • 26. How does copy() work?Beginner
    Evaluates: Slice manipulation.
    Copies elements from src to dst. Returns number of elements copied (min of len(src), len(dst)).
  • 27. What is the map iteration order?Beginner
    Evaluates: Common pitfalls.
    Random. Go explicitly randomizes it to prevent reliance on order.
  • 28. How to delete from a map?Beginner
    Evaluates: Basic operations.
    delete(map, key). It's a no-op if the key doesn't exist.
  • 29. What types can be map keys?Intermediate
    Evaluates: Type system constraints.
    Any comparable type (supports ==). Slices, maps, and functions cannot be keys.
  • 30. Does deleting a key shrink the map memory?Advanced
    Evaluates: Memory management.
    Generally no. Buckets remain allocated. To shrink, you must copy to a new map.
4. Structs, Methods, Pointers
  • 31. Value vs Pointer Receiver?Intermediate
    Evaluates: Method design.
    Use Pointer if modifying state or struct is large. Use Value for small, immutable objects or concurrency safety.
  • 32. Can you call a pointer method on a value?Intermediate
    Evaluates: Syntactic sugar knowledge.
    Yes, if the value is addressable, Go automatically takes the address: (&v).Method().
  • 33. What are struct tags?Beginner
    Evaluates: Serialization knowledge.
    Metadata attached to fields (e.g., `json:"name"`) used by reflection (encoding/json, DB ORMs).
  • 34. Does Go have constructors?Beginner
    Evaluates: Idiomatic Go.
    No. Convention is to use factory functions like NewType().
  • 35. What is struct embedding?Intermediate
    Evaluates: Composition vs Inheritance.
    Including a struct inside another without a name. It promotes fields/methods to the outer struct (Composition).
  • 36. Can you compare two pointers?Beginner
    Evaluates: Pointer logic.
    Yes, they are equal if they point to the same address or are both nil.
  • 37. What is new()?Beginner
    Evaluates: Allocation.
    Allocates zeroed memory for a type and returns a pointer.
  • 38. Can a method be nil-safe?Advanced
    Evaluates: Method dispatch.
    Yes. A method with a pointer receiver can handle being called on a nil pointer (check if p == nil).
  • 39. What is the size of an empty struct struct{}?Intermediate
    Evaluates: Memory optimization.
    Zero bytes. Useful for sets (map[T]struct{}) or signal channels.
  • 40. What is pointer arithmetic?Beginner
    Evaluates: Safety features.
    Go does NOT support pointer arithmetic (like C) unless using the unsafe package.
5. Interfaces & Type System
  • 41. How are interfaces implemented?Beginner
    Evaluates: Implicit vs Explicit.
    Implicitly. A type implements an interface if it defines all the required methods. No implements keyword.
  • 42. What is the empty interface?Beginner
    Evaluates: Dynamic typing.
    interface{} (or any in 1.18+) has no methods, so all types satisfy it. Used for generic-like code.
  • 43. What is a Type Assertion?Intermediate
    Evaluates: Interface usage.
    Extracting the concrete value from an interface: val, ok := i.(Type).
  • 44. What is the "Nil Interface" trap?Advanced
    Evaluates: Deep understanding of interface internals.
    An interface is nil only if (Type=nil, Value=nil). An interface holding a nil pointer (Type=*int, Value=nil) is NOT nil.
  • 45. What is a Type Switch?Intermediate
    Evaluates: Control flow.
    A switch statement that compares types instead of values: switch v.(type) { case int: ... }.
  • 46. Can you embed interfaces?Intermediate
    Evaluates: Interface composition.
    Yes. An interface can include other interfaces (e.g., ReadWriter embeds Reader and Writer).
  • 47. What is the internal structure of an interface?Advanced
    Evaluates: Runtime internals.
    It's a pair of pointers: one to the method table (itable) and one to the data.
  • 48. Best practice: Accept interfaces, return structs?Intermediate
    Evaluates: API design.
    Yes. Be liberal in what you accept (interfaces allow mocking) and conservative in what you return (concrete types allow evolution).
  • 49. Can you cast a struct to an interface?Beginner
    Evaluates: Basic assignment.
    Yes, implicitly, if the struct implements the interface methods.
  • 50. What is interface pollution?Advanced
    Evaluates: Design philosophy.
    Creating interfaces before they are needed. In Go, interfaces should be defined by the consumer (where used), not the producer.
6. Error Handling
  • 51. What is the error type?Beginner
    Evaluates: Basics.
    It's a built-in interface with a single method: Error() string.
  • 52. How to create a custom error?Beginner
    Evaluates: Implementation.
    Use errors.New(), fmt.Errorf(), or create a struct that implements the Error() method.
  • 53. What is error wrapping?Intermediate
    Evaluates: Go 1.13+ features.
    Adding context to an error while preserving the original. Use fmt.Errorf("... %w", err).
  • 54. Difference between errors.Is and errors.As?Intermediate
    Evaluates: Error checking.
    Is checks if an error matches a specific value (sentinel). As checks if it matches a specific type (and casts it).
  • 55. When to use panic?Intermediate
    Evaluates: Best practices.
    Only for unrecoverable errors (e.g., startup config missing). Never for normal control flow.
  • 56. How does recover work?Intermediate
    Evaluates: Panic management.
    It regains control of a panicking goroutine. Must be called inside a defer function.
  • 57. What happens if you don't check an error?Beginner
    Evaluates: Code hygiene.
    The program continues in an undefined/invalid state. Always check errors.
  • 58. What are sentinel errors?Intermediate
    Evaluates: Error patterns.
    Global variables for expected errors, e.g., io.EOF. Checked using errors.Is.
  • 59. How to handle multiple errors?Intermediate
    Evaluates: Advanced handling.
    Use errors.Join (Go 1.20+) or a multierror library to aggregate them.
  • 60. Does defer run if a panic occurs?Intermediate
    Evaluates: Execution flow.
    Yes. Deferred functions are executed during the stack unwinding caused by a panic.
7. Concurrency (Goroutines & Channels)
  • 61. Goroutine vs OS Thread?Intermediate
    Evaluates: Runtime knowledge.
    Goroutines are user-space, lightweight (2KB stack), managed by Go runtime (M:N scheduler). OS threads are heavy (1MB), managed by kernel.
  • 62. Buffered vs Unbuffered Channel?Beginner
    Evaluates: Channel mechanics.
    Unbuffered blocks sender until receiver is ready (sync). Buffered blocks only if full (async).
  • 63. What happens writing to a closed channel?Intermediate
    Evaluates: Safety.
    Panic. Always ensure the sender closes the channel.
  • 64. What happens reading from a closed channel?Intermediate
    Evaluates: Safety.
    Returns the zero value and false immediately. Does not panic.
  • 65. What happens with nil channels?Advanced
    Evaluates: Deadlock awareness.
    Read/Write blocks forever. Closing panics. Useful for disabling cases in select.
  • 66. What is the select statement?Beginner
    Evaluates: Multiplexing.
    Like a switch for channels. It waits for the first channel operation to be ready.
  • 67. How to implement a timeout?Intermediate
    Evaluates: Patterns.
    Use select with time.After() case.
  • 68. What is a goroutine leak?Intermediate
    Evaluates: Resource management.
    When a goroutine starts but never terminates (e.g., blocked on a channel no one writes to).
  • 69. How to stop a goroutine?Intermediate
    Evaluates: Control.
    Signal it via a channel (done channel) or Context cancellation. You cannot kill it externally.
  • 70. What is the default case in select?Intermediate
    Evaluates: Non-blocking ops.
    It executes immediately if no other channel is ready, making the select non-blocking.
8. sync Package & Thread Safety
  • 71. Mutex vs Channel?Intermediate
    Evaluates: Architectural choice.
    Use Channels to pass data/ownership. Use Mutex to protect shared state/cache.
  • 72. What is sync.WaitGroup?Beginner
    Evaluates: Synchronization.
    Waits for a collection of goroutines to finish. Add(), Done(), Wait().
  • 73. What is sync.Once?Intermediate
    Evaluates: Patterns.
    Ensures a function runs exactly once (e.g., singleton initialization). Thread-safe.
  • 74. RWMutex vs Mutex?Intermediate
    Evaluates: Optimization.
    RWMutex allows multiple readers but only one writer. Better for read-heavy workloads.
  • 75. What is sync.Pool?Advanced
    Evaluates: Performance tuning.
    A cache of temporary objects to reduce GC pressure. Items can be removed by GC at any time.
  • 76. What is a Race Condition?Intermediate
    Evaluates: Concurrency bugs.
    When multiple threads access shared memory concurrently and at least one is writing.
  • 77. How to detect race conditions?Beginner
    Evaluates: Tooling.
    Run tests or app with -race flag.
  • 78. What is sync.Map?Advanced
    Evaluates: Specialized structures.
    A concurrent map optimized for append-only or disjoint key sets. Slower than Mutex+Map for general use.
  • 79. What is atomic package?Advanced
    Evaluates: Low-level sync.
    Low-level atomic memory primitives (Add, Load, Store). Faster than Mutex for simple counters.
  • 80. What is a Deadlock?Intermediate
    Evaluates: Bug diagnosis.
    When goroutines are waiting on each other and none can proceed. Go runtime detects some deadlocks.
9. Memory Management & GC
  • 81. Stack vs Heap?Intermediate
    Evaluates: Memory model.
    Stack is fast, local to goroutine. Heap is slower, shared, GC-managed.
  • 82. What is Escape Analysis?Advanced
    Evaluates: Compiler optimization.
    Compiler decides if a variable stays on stack or moves to heap (escapes) based on scope usage.
  • 83. How does Go GC work?Advanced
    Evaluates: Runtime internals.
    Concurrent, Tri-color Mark-and-Sweep. Optimized for low latency (short pauses).
  • 84. What is GOGC?Intermediate
    Evaluates: Tuning.
    Env var to control GC aggressiveness. Default 100 (GC runs when heap doubles).
  • 85. Do pointers affect GC performance?Advanced
    Evaluates: Performance intuition.
    Yes. GC must scan pointer-heavy structures (like maps of pointers). Pointers prevent stack allocation.
  • 86. What is a memory leak in Go?Intermediate
    Evaluates: Debugging.
    Usually goroutine leaks or keeping references to large objects (e.g., substring of large string, slice of large array).
  • 87. Size of a pointer?Beginner
    Evaluates: Basics.
    4 bytes on 32-bit, 8 bytes on 64-bit.
  • 88. What is zero-allocation?Advanced
    Evaluates: Optimization.
    Writing code that avoids heap allocations (using stack, object pooling) to reduce GC pressure.
  • 89. Struct alignment/padding?Advanced
    Evaluates: Memory layout.
    Fields are aligned to memory boundaries. Ordering fields from largest to smallest size saves memory.
  • 90. Does Go have finalizers?Advanced
    Evaluates: Cleanup.
    Yes (runtime.SetFinalizer), but usage is discouraged due to unpredictability.
10. Performance & Optimization
  • 91. How to profile a Go app?Intermediate
    Evaluates: Tooling.
    Use pprof (CPU, Memory, Block, Mutex profiles).
  • 92. String concatenation efficiency?Intermediate
    Evaluates: Optimization.
    Use strings.Builder or bytes.Buffer instead of + for loops to avoid O(N¬≤) allocs.
  • 93. Preallocating slices?Intermediate
    Evaluates: Best practices.
    Always use make([]T, 0, cap) if size is known to avoid resizing allocations.
  • 94. What is inlining?Advanced
    Evaluates: Compiler knowledge.
    Compiler replaces function call with function body. Reduces overhead. Enabled for simple functions.
  • 95. Bounds check elimination?Advanced
    Evaluates: Compiler optimization.
    Compiler removes array bounds checks if it can prove safety, improving loop performance.
  • 96. Reflection performance?Intermediate
    Evaluates: Trade-offs.
    Reflection is slow and prevents compiler optimizations. Avoid in hot paths.
  • 97. False sharing?Advanced
    Evaluates: CPU cache awareness.
    When threads modify independent variables that share a cache line, causing cache invalidation storms. Pad structs to fix.
  • 98. Cgo overhead?Advanced
    Evaluates: FFI.
    Calling C from Go has high overhead (stack switching). Avoid for small/frequent calls.
  • 99. How to benchmark?Beginner
    Evaluates: Testing.
    Write functions starting with BenchmarkXxx(b *testing.B) and run go test -bench=..
  • 100. Map preallocation?Intermediate
    Evaluates: Optimization.
    make(map[K]V, size) hints initial capacity, reducing rehashing overhead.
11. Go Tooling
  • 101. What is go mod?Beginner
    Evaluates: Dependency management.
    The module system. go.mod defines dependencies and versions.
  • 102. What is go sum?Intermediate
    Evaluates: Security.
    Checksum database to verify module integrity and prevent tampering.
  • 103. go build vs go install?Beginner
    Evaluates: Workflow.
    build compiles to current dir. install compiles and moves binary to $GOPATH/bin.
  • 104. What is go vet?Intermediate
    Evaluates: Code quality.
    Static analysis tool to find common errors (e.g., printf args mismatch).
  • 105. What is go fmt?Beginner
    Evaluates: Style.
    Formats code to standard Go style. Eliminates style bike-shedding.
  • 106. What are build tags?Intermediate
    Evaluates: Conditional compilation.
    Comments like //go:build linux to include/exclude files based on OS/Arch/Flags.
  • 107. What is vendoring?Intermediate
    Evaluates: Dependency management.
    Copying dependencies into a local vendor folder for hermetic builds.
  • 108. go get vs go install (modules)?Intermediate
    Evaluates: Modern workflow.
    go get adds dependencies. go install installs binaries (tools).
  • 109. What is gopls?Intermediate
    Evaluates: IDE setup.
    The Go Language Server. Provides IDE features like autocomplete and jump-to-def.
  • 110. How to upgrade dependencies?Beginner
    Evaluates: Maintenance.
    go get -u ./... and go mod tidy.
12. Testing & Benchmarking
  • 111. Table-driven tests?Intermediate
    Evaluates: Idiomatic testing.
    Defining a slice of struct cases (input, expected), loop over them, run t.Run(). The Go way.
  • 112. What is t.Run?Intermediate
    Evaluates: Subtests.
    Runs a subtest. Allows hierarchical test reporting and running specific cases.
  • 113. TestMain?Intermediate
    Evaluates: Setup/Teardown.
    Global setup/teardown for a package. Must call m.Run().
  • 114. How to mock in Go?Intermediate
    Evaluates: Testing strategy.
    Use interfaces. Create a struct that implements the interface with dummy logic.
  • 115. What is httptest?Intermediate
    Evaluates: Web testing.
    Stdlib package to record HTTP responses (ResponseRecorder) and start test servers.
  • 116. What is Fuzzing?Advanced
    Evaluates: Go 1.18+ features.
    Automated testing with random inputs to find crashes/bugs. func FuzzXxx(f *testing.F).
  • 117. How to check test coverage?Beginner
    Evaluates: Quality assurance.
    go test -cover.
  • 118. Parallel testing?Intermediate
    Evaluates: Speed.
    Call t.Parallel() inside the test function.
  • 119. What is iotest?Advanced
    Evaluates: Reader/Writer testing.
    Helpers to test Readers/Writers (e.g., simulate partial reads, errors).
  • 120. Example functions?Beginner
    Evaluates: Documentation.
    Functions starting with Example. They serve as docs and are verified as tests.
13. Standard Library
  • 121. net/http basics?Beginner
    Evaluates: Web dev.
    Production-ready HTTP server/client. http.Handler interface is key.
  • 122. io.Reader / io.Writer?Intermediate
    Evaluates: Abstraction.
    Core interfaces for stream processing. Used everywhere (files, net, buffers).
  • 123. time package pitfalls?Intermediate
    Evaluates: Practical usage.
    Monotonic time vs Wall clock. Formatting uses reference date "Mon Jan 2 15:04:05 2006".
  • 124. encoding/json tags?Beginner
    Evaluates: Serialization.
    `json:"name,omitempty"`. Controls field naming and omission.
  • 125. sort package?Beginner
    Evaluates: Algorithms.
    sort.Slice (custom comparator) or sort.Ints.
  • 126. bufio?Intermediate
    Evaluates: I/O performance.
    Buffered I/O. Reduces syscalls when reading/writing files or network.
  • 127. os/exec?Intermediate
    Evaluates: System interaction.
    Running external commands.
  • 128. sync/atomic?Advanced
    Evaluates: Lock-free sync.
    Atomic operations on integers/pointers.
  • 129. context?Intermediate
    Evaluates: Control flow.
    Deadlines, cancellation, and request-scoped values.
  • 130. math/rand vs crypto/rand?Intermediate
    Evaluates: Security.
    math is pseudo-random (fast, deterministic). crypto is cryptographically secure (slower).
14. Backend Development
  • 131. How to handle HTTP routing?Intermediate
    Evaluates: Web frameworks.
    Stdlib http.ServeMux (Go 1.22+ is good) or libraries like Chi/Gin/Echo.
  • 132. What is Middleware?Intermediate
    Evaluates: Patterns.
    Function wrapping a Handler to add logic (logging, auth) before/after execution.
  • 133. Database connection pooling?Intermediate
    Evaluates: DB performance.
    database/sql handles it automatically. Configure SetMaxOpenConns.
  • 134. Graceful shutdown?Intermediate
    Evaluates: Ops.
    Listen for OS signals (SIGTERM), call server.Shutdown(ctx) to finish active requests.
  • 135. Structured logging?Intermediate
    Evaluates: Observability.
    Use slog (Go 1.21+) or Zap/Logrus. JSON format for machine parsing.
  • 136. How to prevent SQL injection?Beginner
    Evaluates: Security.
    Use parameterized queries (? or $1) in database/sql. Never concat strings.
  • 137. gRPC vs REST?Advanced
    Evaluates: Architecture.
    gRPC (Protobuf) is binary, typed, faster, good for internal services. REST (JSON) is standard, readable, good for public APIs.
  • 138. Configuration management?Intermediate
    Evaluates: Ops.
    Env vars (12-factor app) or Viper (files + env).
  • 139. How to handle JSON?Beginner
    Evaluates: Data format.
    Struct tags + json.Marshal/Unmarshal. Use json.Decoder for streams.
  • 140. Metrics?Intermediate
    Evaluates: Observability.
    Prometheus client library. Expose /metrics endpoint.
15. Context & Cancellation
  • 141. What is context.Background()?Beginner
    Evaluates: Usage.
    The root of the context tree. Used in main, init, and tests.
  • 142. context.TODO()?Beginner
    Evaluates: Usage.
    Placeholder when context is unclear or not yet available.
  • 143. WithCancel?Intermediate
    Evaluates: Control.
    Returns a copy of parent with a new Done channel. Calling cancel closes it.
  • 144. WithTimeout vs WithDeadline?Intermediate
    Evaluates: Time handling.
    Timeout takes a duration (5s). Deadline takes a specific time (2023-01-01...).
  • 145. WithValue pitfalls?Advanced
    Evaluates: Best practices.
    Use only for request-scoped data (TraceID, Auth). Not for optional function params. Keys should be unexported types to avoid collisions.
  • 146. Context propagation?Intermediate
    Evaluates: Flow.
    Pass context as the first argument to functions performing I/O.
  • 147. Storing context in structs?Intermediate
    Evaluates: Anti-patterns.
    Avoid it. Context should flow through function calls. Exception: request structs.
  • 148. HTTP Request Context?Intermediate
    Evaluates: Web dev.
    req.Context(). Cancelled when client disconnects.
  • 149. Database Context?Intermediate
    Evaluates: DB.
    db.QueryContext(ctx, ...). Cancels query if context expires.
  • 150. Cancellation propagation?Advanced
    Evaluates: Tree structure.
    Cancelling a parent context cancels all its children.
16. Design Patterns in Go
  • 151. Functional Options?Intermediate
    Evaluates: API design.
    Pattern for optional config args: NewServer(addr, WithTLS(), WithLogger()).
  • 152. Builder Pattern?Intermediate
    Evaluates: Object creation.
    Less common in Go (Func Options preferred), but used for complex object construction.
  • 153. Singleton?Beginner
    Evaluates: Sync.
    Use sync.Once to ensure lazy initialization happens exactly once.
  • 154. Factory?Beginner
    Evaluates: Creation.
    Simple functions like NewThing() returning an interface or struct.
  • 155. Adapter?Intermediate
    Evaluates: Interfaces.
    Converting one interface to another (e.g., http.HandlerFunc adapts a func to Handler).
  • 156. Decorator?Intermediate
    Evaluates: Composition.
    Wrapping a type to add behavior. Common in Middleware (wrapping http.Handler).
  • 157. Observer?Advanced
    Evaluates: Events.
    Often implemented with Channels (pub/sub).
  • 158. Strategy?Intermediate
    Evaluates: Behavior.
    Defining behavior via an interface field, swappable at runtime.
  • 159. Worker Pool?Intermediate
    Evaluates: Concurrency patterns.
    Fixed number of goroutines processing jobs from a channel. Limits concurrency.
  • 160. Fan-in / Fan-out?Intermediate
    Evaluates: Concurrency patterns.
    Fan-out: Distribute work to multiple workers. Fan-in: Collect results into one channel.
17. Generics (Go 1.18+)
  • 161. What are Type Parameters?Intermediate
    Evaluates: Syntax.
    func Index[T comparable](s []T, x T). Allows writing code for multiple types.
  • 162. What is a Constraint?Intermediate
    Evaluates: Interfaces.
    An interface defining the set of allowed types (e.g., any, comparable).
  • 163. What is any?Beginner
    Evaluates: Alias.
    Alias for interface{}.

22. Go Concurrency Patterns
  • 256. Worker PoolIntermediate

    Concept: A fixed number of workers pull tasks from a shared channel. Useful for controlling concurrency and resource usage.

    func worker(id int, jobs <-chan int, results chan<- int) {
    for j := range jobs {
        results <- j * 2 // Process job
    }
}

func main() {
    jobs := make(chan int, 100)
    results := make(chan int, 100)

    // Start 3 workers
    for w := 1; w <= 3; w++ {
        go worker(w, jobs, results)
    }

    // Send 5 jobs
    for j := 1; j <= 5; j++ {
        jobs <- j
    }
    close(jobs)

    // Collect results
    for a := 1; a <= 5; a++ {
        <-results
    }
}
  • 257. Fan-Out / Fan-InAdvanced

    Fan-Out: Multiple functions read from the same channel to distribute work. Fan-In: Multiple channels are multiplexed into a single output channel.

    func merge(cs ...<-chan int) <-chan int {
    var wg sync.WaitGroup
    out := make(chan int)

    // Start an output goroutine for each input channel
    output := func(c <-chan int) {
        defer wg.Done()
        for n := range c { out <- n }
    }

    wg.Add(len(cs))
    for _, c := range cs {
        go output(c)
    }

    // Close out channel when all inputs are done
    go func() {
        wg.Wait()
        close(out)
    }()
    return out
}

    Concept: A series of stages connected by channels, where each stage processes data and passes it to the next.
